# Financial Advisor Multi-Agent AI

This module provides a multi-agent financial advisory assistant built with the `google-adk` framework. It features a master orchestration agent connecting multiple downstream "expert" subagents to guide users through a structured financial analysis and trading pipeline.

## 🏗 Architecture

The system uses a **Hierarchical Multi-Agent Architecture**.

### The Orchestrator
The root agent is the **Financial Coordinator** (`financial_coordinator`). Its role is conversational flow management and delegation. It:
- Greets the user and presents financial disclaimers.
- Collects initial input (like ticker symbols, risk tolerance, and investment timelines).
- Delegates complex tasks to the sub-agents natively using the `AgentTool` mechanism in ADK.
- Compiles the outputs of each step, ensuring state is passed along securely.

### The Subagents
There are four specialized expert subagents located in the `financial_advisor/sub_agents` directory:

1. **Data Analyst (`data_analyst`)**
   - **Responsibility:** Retrieves and compiles a comprehensive data analysis for a specific market ticker.
   - **Input:** Ticker symbol (e.g. AAPL, GOOGL).
2. **Trading Analyst (`trading_analyst`)**
   - **Responsibility:** Develops tailored trading strategies.
   - **Input:** Market data analysis (from step 1), user risk attitude, and investment period.
3. **Execution Analyst (`execution_analyst`)** 
   - **Responsibility:** Defines an optimal execution plan (order types, timing, cost implications).
   - **Input:** Proposed trading strategies (from step 2) and user preferences.
4. **Risk Analyst (`risk_analyst`)**
   - **Responsibility:** Evaluates overall portfolio risk and highlights potential misalignments or concentrated vulnerabilities.
   - **Input:** Market data, trading strategies, execution plan, and the user's risk horizon.

*Note: All agents utilize the local `openai/gpt-oss:20b` litellm instance connected via Ollama / local inference endpoints.*

## 🚀 Usage

You can run this multi-agent setup utilizing the primary `google-adk` runner interfaces. 

To spin up the web interface:
```bash
uv run adk web financial_advisor
```

### The Interactive Flow
Once running, the interaction generally proceeds step-by-step:
1. The orchestrator will introduce itself and ask you for a **Market Ticker**.
2. It will pass that ticker to the **Data Analyst** and visualize the return.
3. It will then ask you for your **Risk Attitude** (conservative, moderate, aggressive) and **Investment Period** (short-, medium-, or long-term).
4. The orchestrator passes this context down the pipeline to the **Trading Analyst**, **Execution Analyst**, and finally the **Risk Analyst**, generating a holistic multi-faceted markdown report.
